\section{Introdução}
    \frame
    {
        \frametitle{Introdução}
        \begin{itemize}
            \item Tipicamente, as LPSs são limitados a um contexto intra-organizacional
            \item Quando uma companhia decide expandir a LPS para além dos seus limites, 
            é feita a transição da LPS para um ecosistema de software
        \end{itemize}
    }
    \frame
    {
        \frametitle{Motivos para a transição}
        \begin{itemize}
            \item A companhia não é capaz de, sozinha, construir todo o sistema
            em tempo hábil com os recursos disponíveis
            \item Customização em massa necessidade de um significante investimento em P\&D
        \end{itemize}
    }
    \frame
    {
        \frametitle{Argumentos que explicam essa tendência}
        \begin{itemize}
            \item Aumentar o valor da oferta básica para os usuários existentes
            \item Aumentar a atratividade para novos usuários
            \item Aumentar a "stickiness" (viscosidade?) da plataforma (???)
            \item Acelerar a inovação através da inovação aberta no ecossistema 
            \begin{itemize}
                \item Colaboração com parceiros no ecossistemas para compartilhar
                custos da inovação
                \item Tornar parte da plataforma uma funcionalidade desenvolvida
                por parceiros
            \end{itemize}
            \item Dimininuir o TCO através da comoditização de funcionalidades, 
            compartilhando a manutenção com parceiros
        \end{itemize}
    }
    
\section{Taxonomia dos ecossistemas de software}
    \subsection{Introdução}
    \frame
    {
        \frametitle{Taxonomia dos ecossistemas de software}
        \begin{itemize}
            \item Ecossistema natural
            \item Ecossistema social
            \item Ecossistema de software
        \end{itemize}
    }
    \frame
    {
        \frametitle{Ecossistema natural}
        \begin{quote}
            Ecossistema natural é o conjunto de todas plantas, animais e micro-organismos
            que vivem em uma determinada região, juntamente com os fatores abióticos que 
            influenciam esse ambiente (Wikipédia)
        \end{quote}
    }
    \frame
    {
        \frametitle{Ecossistema humano}
        \begin{quote}
             Ecossistema humano consiste de atores, as interconexões entres atores, 
             as atividades feitas por essas atores e as transações através dessas conexões 
             concernentes a fatores físicos e não-físicos 
            \end{quote}
        \begin{itemize}
            \item Ecossistema comercial: os atores são empresários, fornecedores
                    e clientes, os fatores são os serviços e as transações incluem 
                    não só transações financeiras, mas também compartilhamento de 
                    informações e conhecimentos, contatos pré e pós-venda etc.
           \item Ecossistema social: consiste das pessoas, suas relações 
                    sociais e a trocas de várias formas de informação 
        \end{itemize}
    }
    \frame
    {
        \frametitle{Ecossistema de software}
        \begin{quote}
            Um ecossistema de software consiste de um conjuto de soluções de software
            que \textbf{possibilitam}, \textbf{auxiliam} e \textbf{automatizam} as 
            atividades e transações feitas pelos
            atores por motivos sociais ou de negócios e pelas organizações que fornecem
            essas soluções
        \end{quote}
    }
    \frame
    {
        \frametitle{Exemplos}
        \begin{itemize}
            \item Auxiliam: a tarefa é simplificada, sendo necessário menor esforço 
            para ser concluída 
            \item Automatizam: a tarefa é feita por soluções de software, sem 
            interação humana
            \item Possibitam: tornam possível tarefas antes impossíveis (ou economicamente 
            inviável)
        \end{itemize}
    }

    \subsection{Ecossistemas centrados em sistemas operacionais}
    \frame
    {
        \frametitle{Ecossistemas centrados em sistemas operacionais}
        \begin{itemize}
            \item Características
            \begin{itemize}
                \item Independentes de domínio e assumem que terceiros irão desenvolver
                aplicações 
                \item Para desktop e mobile são instalados em cada dispositivo que 
                deseje executar as aplicações
                \item Otimizam a implantação de aplicações e oferecem um pequeno suporte
                para a integração dessas aplicações
                \item Seu criadores oferecem ferramentas que facilitam o desenvolvimento
                de aplicações
            \end{itemize}
        \end{itemize}
    }
    \frame
    {
        \frametitle{Ecossistemas centrados em sistemas operacionais}
        \begin{itemize}
            \item Fatores de sucesso
            \begin{itemize}
                \item O sucesso é definido pelo número de aplicações que executam 
                no sistema
                \item Possibilidade de ser extendido pelos desenvolvedores
                \item Número de usuários que usam o S.O. e são acessíveis pelos 
                desenvolvedores para monetização
            \end{itemize}
        \end{itemize}
    }
    \frame
    {
        \frametitle{Ecossistemas centrados em sistemas operacionais}
        \begin{itemize}
            \item Desafios
            \begin{itemize}
                \item Complexidade de diferentes configurações de hardware
                \item Depois do sucesso financeiro do Windows, a indústria não
                deseja mais uma única plataforma na web ou nos dispositivos móveis 
            \end{itemize}
        \end{itemize}
    }
    
    \subsection{Ecossistemas centrados em aplicações}
    \frame
    {
        \frametitle{Ecossistemas centrados em aplicações}
        \begin{itemize}
            \item Características
            \begin{itemize}
                \item Surgem a partir do sucesso de outras aplicações online
                \item O fornecedor da aplicação disponibiliza meios para que a 
                transição entre a sua aplicação e a de terceiros seja transparente
                para o usuário
                \item Desenvolvedores de aplicações de terceiros estendem funcionalidades
                específicas de domínios fornecidas pela plataforma
                \item A integração a aplicação e extensões é facilitada através de dados, 
                workflow e experiência do usuário
            \end{itemize}
        \end{itemize}
    }
    \frame
    {
        \frametitle{Ecossistemas centrados em aplicações}
        \begin{itemize}
            \item Fatores de sucesso
            \begin{itemize}
                \item Grande número de usuários (ou expectativa) que desejam estender
                a aplicação com novas funcionalidades
                \item O criador da plataforma deve simplificar a contribuição
                por terceiros
                \item Foco em fornecer acesso aos dados, provendo soluções para
                estender o modelo de dados e workflow, e integrar as aplicações
                \item As aplicações de terceiros precisam de cliente e o provedor 
                da plataforma deve fornecer um canal entre os clientes e as aplicações
            \end{itemize}
        \end{itemize}
     }
     \frame
     {
     \frametitle{Ecossistemas centrados em aplicações}
        \begin{itemize}
            \item Desafios
            \begin{itemize}
                \item Conflitos entre a estrágia do produto e a estratégia da aplicação
                \item Estabelecer um modelo viável de negóciso para aplicações de 
                terceiros parece ser uma meta ilusória
            \end{itemize}
        \end{itemize}
    }
    
    \subsection{Ecossistemas de programação para o usuário final}
    \frame
    {
        \frametitle{Ecossistemas de programação para o usuário final}
        \begin{itemize}
            \item Características
            \begin{itemize}
                \item Focado em desenvolvedores de aplicações com bom entendimento
                do domínio, mas sem conhecimento em computação
                \item Os ambientes de desenvolvimento para web costumam utilizar
                uma arquitetura pipe-e-filtros
                \item Do ponto de vista da engenharia de software, geralmente as
                as aplicações são criadas através da composição de blocos pré-definidos
            \end{itemize}
        \end{itemize}
    }
    \frame
    {
        \frametitle{Ecossistemas de programação para o usuário final}
        \begin{itemize}
            \item Fatores de sucesso
            \begin{itemize}
                \item Valor percebido pelo usuário pela criação de uma solução
                única, específica para o cliente
                \item Mecanismos que permitam os usuários compartilharem suas soluções 
                para que outros usuários possam também adotá-las, ajustá-las e usá-las
            \end{itemize}
        \end{itemize}
    }
    \frame
    {
        \frametitle{Ecossistemas de programação para o usuário final}
        \begin{itemize}
            \item Desafios
            \begin{itemize}
                \item Modelar o domínio da aplicação em termos que usuário final 
                irá compreende-lo com um mínimo de esforço e fazer com que a criação
                seja intuitiva e também requeira o mínimo de entendimento
                \item O domínio do ecossistema deve ser suficientemente estável 
            \end{itemize}
        \end{itemize}
    }
