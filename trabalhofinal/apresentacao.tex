\section{Estudo de caso}
    \subsection{O projeto Froid - Fortunes for Android}
    \frame
    {
        \frametitle{Feature Model}
        \begin{figure}[t]
		    \includegraphics[scale=0.4]{img/fortunes-fm}
		\end{figure}
    }
    \frame
    {
        \frametitle{Feature Model}
        \begin{itemize}
            \item Versão: poderia haver duas versões da aplicação - online e offine.
            A primeira com acesso a Internet e a segunda sem. Além disso cada uma 
            teria recursos próprios
            \item Fonte das fortunes: as fortunes podem ser lidas de um banco de
            dados local ou da Internet, através de web service. Há ainda a opção
            delas serem "fixas" para testar as demais funcionalidades da aplicação
            \item Compartilhamento de fortunes: as fortunes podem ser compartilhadas
            por qualquer meio para o qual exista uma aplicação instalada no dispositivo.
            Aparentemente, não existe uma forma de filtrar de filtrar por quais
            meios a mensagem será compartilhada
        \end{itemize}
    }
    \frame
    {
        \frametitle{Feature Model}
        \begin{itemize}
            \item Posição da tela: a fortune pode ser lida com o aparelho tanto na
            posição retrato quanto na paisagem
            \item Fundo: fui sugerido a possibilidade da aplicação ter uma imagem
            de fundo, mas ainda não foi implementado
            \item Categorias: as frases podem ser agrupadas por categorias
        \end{itemize}
    }
    \subsection{Mecanismos de controle das variabilidades}
    \frame
    {
        \frametitle{Ponto de variação: Fonte das fortunes}
        \begin{itemize}
            \item Mecanismo: Polimorfismo 
            \item Cada variação é uma implementação da interface FortuneProvider
            \item Implementações atuais: FortuneProviderDatabase, FortuneProviderService 
            e FortuneProviderStatic
            \item A mudança deve ser feito em tempo de compilação
		\end{itemize}
    }
    \frame
    {
        \frametitle{Ponto de variação: Orientação da tela}
        \begin{itemize}
            \item Mecanismo: arquivos de layout definidos em XML's + 
            armazenamento de estado atual 
            \item A aplicação poderá ter dois layout: um para retrato e outro para 
            paisagem
            \item O Android desenha as telas com base nesse arquivo, selecionando 
            o arquivo apropriado automaticamente, no entanto, deverá ser de responsabilidade
            da aplicação atualização das informações exibidas. 
		\end{itemize}
    }
